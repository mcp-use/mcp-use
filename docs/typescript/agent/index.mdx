`MCPAgent` connects an LLM to one or more MCP servers. It discovers the servers' tools, lets the LLM call them to complete a task, and returns the result. You provide an LLM, and the agent handles tool selection, multi-step execution, and conversation memory.

## Key features

- **LLM tool calling**: Connect a supported LLM to MCP server tools and run a multi-step loop until the task is done.
- **Multiple providers**: OpenAI, Anthropic, Google, and Groq. See [LLM Integration](/typescript/agent/llm-integration).
- **Streaming**: Stream each step or low-level events as the agent runs. See [Streaming](/typescript/agent/streaming).
- **Conversation memory**: Remember earlier turns across `run` calls, on by default. See [Memory Management](/typescript/agent/memory-management).
- **Multi-server**: Connect to several servers and let a [Server Manager](/typescript/agent/server-manager) pick the right tools.

## Installation

Install `mcp-use` and one LangChain provider package for the LLM you want to use.

<CodeGroup>
```bash npm
npm install mcp-use
```
```bash pnpm
pnpm add mcp-use
```
```bash yarn
yarn add mcp-use
```
</CodeGroup>

<CodeGroup>
```bash OpenAI
npm install @langchain/openai
```
```bash Anthropic
npm install @langchain/anthropic
```
```bash Google
npm install @langchain/google-genai
```
```bash Groq
npm install @langchain/groq
```
</CodeGroup>

Set the matching API key in your environment: `OPENAI_API_KEY`, `ANTHROPIC_API_KEY`, `GOOGLE_API_KEY`, or `GROQ_API_KEY`.

## Quick start

This agent connects to the filesystem MCP server and answers a question about the current directory. It uses OpenAI, so set `OPENAI_API_KEY` before running.

Pass a `"provider/model"` string and your server config. The agent builds the LLM and MCP client for you.

```typescript
import { MCPAgent } from "mcp-use"

const agent = new MCPAgent({
  llm: "openai/gpt-4o",
  mcpServers: {
    filesystem: {
      command: "npx",
      args: ["-y", "@modelcontextprotocol/server-filesystem", "./"],
    },
  },
})

const result = await agent.run({ prompt: "What is in the current folder?" })
console.log(result)

await agent.close()
```

`agent.run()` returns the final answer as a string. `maxSteps` (default `5`) caps how many tool-calling steps the agent takes before it must answer; raise it for complex tasks with `new MCPAgent({ ..., maxSteps: 10 })` or per call with `agent.run({ prompt, maxSteps: 10 })`. Always call `agent.close()` to shut down the server connections.

## Core capabilities

### Structured output

Pass a Zod `schema` to `run` to get a typed object back instead of a string.

```typescript
import { z } from 'zod'

const result = await agent.run({
  prompt: 'Analyze the file structure',
  schema: z.object({
    totalFiles: z.number(),
    fileTypes: z.array(z.string()),
    largestFile: z.string()
  })
})

console.log(result.totalFiles) // typed as number
```

See [Structured Output](/typescript/agent/structured-output) for details.

### Streaming

Iterate over `agent.stream` to receive each step as the agent runs.

```typescript
for await (const step of agent.stream({ prompt: 'Write a report' })) {
  console.log(`Tool: ${step.action.tool}`)
}
```

`agent.prettyStreamEvents` and `agent.streamEvents` give formatted and low-level output. See [Streaming](/typescript/agent/streaming).

### Memory

Conversation memory is on by default, so the agent remembers earlier turns across `run` calls.

```typescript
await agent.run({ prompt: "Hello, my name is Alice" })
await agent.run({ prompt: "What's my name?" }) // the agent remembers Alice

agent.clearConversationHistory() // reset the history
```

Pass `memoryEnabled: false` to `new MCPAgent({ ... })` to disable it. See [Memory Management](/typescript/agent/memory-management).

## API reference

`MCPAgent` accepts options in one of two modes:

```typescript
// Explicit mode: you build the LLM and client
interface ExplicitModeOptions {
  llm: BaseChatModel                          // a LangChain chat model, e.g. new ChatOpenAI(...)
  client?: MCPClient
  connectors?: BaseConnector[]
  maxSteps?: number                           // default 5
  memoryEnabled?: boolean                     // default true
  systemPrompt?: string
  useServerManager?: boolean                  // let the agent manage multiple servers
  verbose?: boolean
}

// Simplified mode: the agent builds the LLM and client from your config
interface SimplifiedModeOptions {
  llm: string                                 // "provider/model", e.g. "openai/gpt-4o"
  mcpServers: Record<string, MCPServerConfig>
  llmConfig?: LLMConfig                        // temperature, maxTokens, apiKey, ...
  maxSteps?: number                           // default 5
  memoryEnabled?: boolean                      // default true
  systemPrompt?: string
}
```

Methods and run options:

```typescript
class MCPAgent {
  constructor(options: ExplicitModeOptions | SimplifiedModeOptions)

  run(options: RunOptions): Promise<string>
  run<T>(options: RunOptions<T>): Promise<T>             // with a schema
  stream(options: RunOptions): AsyncGenerator<AgentStep, string>
  streamEvents(options: RunOptions): AsyncGenerator      // low-level LangChain events
  clearConversationHistory(): void
  close(): Promise<void>
}

interface RunOptions<T = string> {
  prompt: string
  maxSteps?: number
  schema?: ZodSchema<T>            // return a typed object instead of a string
  signal?: AbortSignal             // cancel the run
  externalHistory?: BaseMessage[]
}
```
