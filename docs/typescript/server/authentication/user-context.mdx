When OAuth is configured and a request is authenticated, the tool callback's `context.auth` is populated with the authenticated user's information. This enables you to build personalized, user-aware tools and implement access control patterns.

`context.auth` is an `AuthInfo` object. The user's profile fields live on `context.auth.user`. It is `undefined` when no OAuth middleware ran or the request was not authenticated, so guard with `if (!context.auth)` before using it:

```typescript
import { z } from 'zod'
import { text, error } from 'mcp-use/server'

server.tool({
  name: 'create-document',
  schema: z.object({
    title: z.string(),
    content: z.string()
  }),
  cb: async ({ title, content }, context) => {
    // Guard: auth is only set when the request was authenticated
    if (!context.auth) {
      return error('Unauthorized')
    }

    // Access the authenticated user (context.auth.user is a UserInfo)
    const { user } = context.auth

    // User info available:
    console.log('User ID:', user.userId)         // Unique user identifier
    console.log('Email:', user.email)            // User email
    console.log('Name:', user.name)              // Display name
    console.log('Roles:', user.roles)            // User roles
    console.log('Permissions:', user.permissions) // Permissions

    // Create document with user context
    const doc = await db.documents.create({
      title,
      content,
      createdBy: user.userId,
      createdByName: user.name
    })

    return text(`Document created by ${user.name}`)
  }
})
```

## Auth Info Type

`context.auth` is an `AuthInfo` object. The user's profile fields live on `context.auth.user` (a `UserInfo`), while scopes and permissions are top-level:

```typescript
interface AuthInfo {
  user: UserInfo                       // Authenticated user profile (see below)
  payload: Record<string, unknown>     // Raw JWT claims
  accessToken: string                  // The bearer token
  scopes: string[]                     // OAuth scopes granted
  permissions: string[]                // Effective permissions
}
```

The `context.auth.user` object (`UserInfo`) contains the following fields:

```typescript
interface UserInfo {
  userId: string              // Unique user identifier (from 'sub' claim)
  email?: string              // User email
  name?: string               // Full name
  username?: string           // Username
  nickname?: string           // Nickname or display name
  picture?: string            // Profile picture URL
  roles?: string[]            // User roles
  permissions?: string[]      // User permissions

  // Provider-specific fields
  [key: string]: any          // Additional claims from provider
}
```

### Standard Fields

#### userId (required)
The unique identifier for the user, extracted from the JWT `sub` claim. Always present and guaranteed to be unique.

```typescript
const userId = context.auth.user.userId // "auth0|507f1f77bcf86cd799439011"
```

#### email
The user's email address. Available when the `email` scope is granted.

```typescript
const email = context.auth.user.email // "user@example.com"
```

#### name
The user's full name or display name.

```typescript
const name = context.auth.user.name // "John Doe"
```

#### username
The user's username, if different from email.

```typescript
const username = context.auth.user.username // "johndoe"
```

#### picture
URL to the user's profile picture.

```typescript
const picture = context.auth.user.picture // "https://avatar.example.com/user.jpg"
```

### Authorization Fields

#### roles
Array of role names assigned to the user. Used for role-based access control (RBAC).

```typescript
const roles = context.auth.user.roles // ["editor", "reviewer"]

// Check if user has admin role
if (context.auth.user.roles?.includes('admin')) {
  // Allow admin action
}
```

#### permissions
Array of specific permissions granted to the user. More granular than roles.

```typescript
const permissions = context.auth.user.permissions
// ["documents:read", "documents:write", "users:read"]

// Check for specific permission
if (context.auth.user.permissions?.includes('documents:delete')) {
  // Allow document deletion
}
```

#### scopes
OAuth scopes granted during authentication. Unlike the fields above, `scopes` lives on `context.auth` directly (an `AuthInfo` field), not on `context.auth.user`.

```typescript
const scopes = context.auth.scopes // ["openid", "profile", "email"]

// Check if user granted email scope
if (context.auth.scopes?.includes('email')) {
  // Access email information
}
```


## Extracting Custom Claims

If you need to extract custom claims differently, use the `getUserInfo` function in your OAuth configuration:

```typescript
import { MCPServer, oauthAuth0Provider } from 'mcp-use/server'

const server = new MCPServer({
  oauth: oauthAuth0Provider({
    domain: 'your-tenant.auth0.com',
    audience: 'https://your-api.example.com',
    
    // Custom user info extraction
    getUserInfo: (payload) => {
      return {
        userId: payload.sub,
        email: payload.email,
        name: payload.name,
        // Extract custom claims
        organizationId: payload['https://myapp.com/org_id'],
        roles: payload['https://myapp.com/roles'] || [],
        permissions: payload.permissions || [],
        // Include all other claims
        ...payload
      }
    }
  })
})
```
