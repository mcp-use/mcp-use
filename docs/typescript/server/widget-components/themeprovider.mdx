The `ThemeProvider` component manages theme by toggling the `light`/`dark` class, setting a `data-theme` attribute, and (optionally) the `color-scheme` style on the document root. It prioritizes an explicit theme from the host context, falling back to system preference when the host does not provide one.

<Tip>
  You can use `<McpUseProvider />` to automatically include the `<ThemeProvider />` component.
</Tip>

## Import

```typescript
import { ThemeProvider } from 'mcp-use/react';
```

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `React.ReactNode` | **required** | The widget content to wrap |
| `colorScheme` | `boolean` | `true` | When true, sets `document.documentElement.style.colorScheme` to the active theme so CSS `light-dark()` resolves correctly. |

## Theme Priority

The provider uses the following priority:

1. **Host theme** (from `useWidget().hostContext.theme`) - When the host provides an explicit `"dark"` or `"light"` value
2. **System preference** (`prefers-color-scheme: dark`) - Fallback when the host does not provide an explicit theme

## Basic Usage

```tsx
import { ThemeProvider } from 'mcp-use/react';

function MyWidget() {
  return (
    <ThemeProvider>
      <div>My widget content</div>
    </ThemeProvider>
  );
}
```


## Theme Synchronization

The provider:

- **Syncs with ChatGPT**: Automatically matches ChatGPT's theme when running in Apps SDK
- **Listens to system changes**: Updates when user changes system theme preference
- **Prevents flash**: Uses `useLayoutEffect` to apply theme synchronously before browser paint
