Content Security Policy (CSP) controls which domains your widgets can fetch from, load scripts and styles from, and embed. Widgets run in sandboxed iframes, so CSP must explicitly allow any external resources.

## Automatic configuration

When `baseUrl` is set (via `MCPServer` constructor or `MCP_URL` environment variable), mcp-use automatically configures CSP:

```typescript
const server = new MCPServer({
  name: "my-server",
  version: "1.0.0",
  baseUrl: process.env.MCP_URL, // Required for production
});
```

This ensures widget URLs use the correct domain, CSP includes your server origin, and deployments behind proxies keep working.

The **server origin** is auto-injected into each widgetŌĆÖs effective `connectDomains`, `resourceDomains`, and `baseUriDomains` ŌĆö you do not add it manually for your own API host in the common case.

## Inline JSX: `_csp` and component `.config`

For the **primary** pattern, set CSP on the JSX returned from the tool with **`_csp`**, and/or attach static CSP to the component with **`defineWidget`** / **`.config`** (see [MCP Apps](./mcp-apps)).

**Per tool call (JSX props)**

```tsx
/** @jsxImportSource mcp-use/jsx */
import { text } from "mcp-use/server";
import ChartView from "./components/ChartView";

return (
  <ChartView
    data={series}
    _output={text("Chart ready")}
    _csp={{
      connectDomains: ["https://api.example.com"],
      resourceDomains: ["https://cdn.example.com"],
      frameDomains: ["https://trusted-embed.com"],
      baseUriDomains: ["https://myserver.com"],
      redirectDomains: ["https://oauth.provider.com"],
    }}
  />
);
```

**Per component (static)**

```tsx
import { defineWidget } from "mcp-use/react";

export default defineWidget(ChartViewInner, {
  csp: { connectDomains: ["https://api.example.com"] },
});
```

Merge order: **defaults ŌåÆ component `.config` ŌåÆ `_csp` on the JSX** for that invocation.

## Development mode enrichment

In development, mcp-use may **extend** CSP so local workflows work (for example WebSocket or eval used by the dev server). Treat production CSP as authoritative; use the InspectorŌĆÖs **Widget-Declared** CSP mode to validate what you declare.

## Field reference

| MCP Apps (camelCase) | Apps SDK (snake_case) | Description |
| -------------------- | --------------------- | ----------- |
| `connectDomains` | `connect_domains` | Domains for fetch, XHR, WebSocket |
| `resourceDomains` | `resource_domains` | Scripts, styles, images, fonts |
| `baseUriDomains` | `base_uri_domains` | Base URI allowlist |
| `frameDomains` | `frame_domains` | Nested iframes |
| `redirectDomains` | `redirect_domains` | Redirects (ChatGPT / `openExternal`) |
| `scriptDirectives` | `script_directives` | Extra script CSP directives |
| `styleDirectives` | `style_directives` | Extra style CSP directives |

mcp-use maps unified camelCase config to MCP Apps **`_meta.ui.csp`** and to ChatGPT **`openai/widgetCSP`** automatically.

## Environment variables

```env
MCP_URL=https://myserver.com
CSP_URLS=https://api.example.com,https://cdn.example.com
```

- **`MCP_URL`** ŌĆö Public origin for assets and CSP base configuration.
- **`CSP_URLS`** ŌĆö Optional comma-separated extra domains merged into widget CSP (useful when static assets live on another origin).

## Static deployments

When widgets are served from static storage while the MCP server runs elsewhere, set **`MCP_URL`**, **`MCP_SERVER_URL`**, and **`CSP_URLS`** as described in [Supabase deployment](./deployment/supabase).

## Inspector debugging

The Inspector provides a **CSP mode** toggle:

- **Permissive** ŌĆö Relaxed CSP for debugging.
- **Widget-Declared** ŌĆö Enforces declared CSP (closer to production).

CSP violations appear in the browser console. See [Debugging ChatGPT Apps](/inspector/debugging-chatgpt-apps).

## Legacy: `widgetMetadata` and `server.uiResource`

Older docs used `widgetMetadata.metadata.csp` or `server.uiResource({ metadata: { csp } })`. The modern equivalents are **`_csp`** / **`.config`** above. For side-by-side migration steps, see **[Migrating from resources/ and widgetMetadata](./migration-from-resources)**.
