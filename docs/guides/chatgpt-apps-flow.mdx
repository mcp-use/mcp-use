# ChatGPT and MCP Apps widgets

## What to use today

1. **[MCP Apps overview](/typescript/server/mcp-apps)** — Inline JSX (`/** @jsxImportSource mcp-use/jsx */`), `_output`, `_csp`, `streamable()`, `ToolRef`, `ctx.client.supportsApps()`.
2. **[Creating an MCP Apps server](/typescript/server/creating-mcp-apps-server)** — Step-by-step server + component layout.
3. **[Migrating from resources/ and widgetMetadata](/typescript/server/migration-from-resources)** — If you have older `resources/` or `widgetMetadata` code.

mcp-use turns your JSX return into:

- **MCP Apps** metadata (`text/html;profile=mcp-app`, `_meta.ui`, CSP in camelCase).
- **ChatGPT / Apps SDK** metadata (`text/html+skybridge`, `openai/widgetCSP`, etc.).

You write **one** widget path; hosts negotiate the protocol.

## How this relates to ChatGPT

- **ChatGPT** embeds the widget iframe and exposes `window.openai` APIs.
- **MCP Apps** hosts (Claude, Goose, Inspector, etc.) use JSON-RPC over `postMessage`.

Your **React code** stays the same: `useWidget`, `useCallTool`, `McpUseProvider`, etc. The bridge abstracts protocol differences.
