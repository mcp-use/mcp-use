mcp-use is a fullstack MCP (Model Context Protocol) framework for Python and TypeScript.

**Install:**
- Python: `pip install mcp-use` (agent also requires a LangChain LLM provider, e.g. `pip install langchain-openai`)
- TypeScript: `npm install mcp-use`

**Core classes:**
- `MCPServer`  -  build an MCP server that exposes tools, resources, and prompts
- `MCPClient`  -  connect to one or more MCP servers programmatically
- `MCPAgent`  -  LLM-powered agent that uses MCPClient to call tools (requires LangChain)

**Key imports:**
- Python server: `from mcp_use import MCPServer`
- Python client/agent: `from mcp_use import MCPClient, MCPAgent`
- TypeScript server: `import { MCPServer } from "mcp-use/server"`
- TypeScript client/agent: `import { MCPClient, MCPAgent } from "mcp-use"`

**TypeScript tool definition:**

## What is MCP?

The **Model Context Protocol (MCP)** is an open standard for connecting AI models to external tools, data sources, and services. Instead of custom integrations for every API, MCP provides a universal interface that any LLM host can speak.

## Architecture

MCP uses a client-server model with three roles:

| Role | Description |
|------|-------------|
| **MCP Host** | The AI-powered application  -  your product, Claude Desktop, Cursor |
| **MCP Client** | Runs inside the host; manages communication with MCP servers |
| **MCP Server** | Exposes tools, resources, and prompts to connected clients |

```
MCP Host
└─ MCP Client ── MCP Server A (GitHub)
              ├─ MCP Server B (Database)
              └─ MCP Server C (Custom API)
```

The client queries each server for available capabilities, then routes model requests to the right server.

## Server Primitives

MCP servers expose three primitives:

| Primitive | Controlled by | Description | Example |
|-----------|--------------|-------------|---------|
| **Tools** | Model | Functions the LLM calls to take actions | API calls, database writes |
| **Resources** | Application | Read-only data the client can access | File contents, query results |
| **Prompts** | User | Reusable prompt templates | Slash commands, menu options |

### Tools

Functions the model can invoke. The server executes them and returns results  -  API calls, computations, file writes, etc.

### Resources

Read-only data sources. Each resource has a unique URI (`file:///example.txt`, `db://users/123`), a display name, and optional MIME type. The client fetches them on demand.

### Prompts

Templated instructions managed by the server. Useful for consistency and for providing the model with structured context.

## Client Primitives

MCP clients can implement additional primitives that reverse the flow:

| Primitive | Flow | Description |
|-----------|------|-------------|
| **Sampling** | Server → LLM | Server requests an LLM completion from the client |
| **Elicitation** | Server → User | Server asks the user for input mid-workflow |
| **Roots** | Server → Filesystem | Server requests access to specific files or directories |

## mcp-use

mcp-use is a fullstack MCP framework for building **MCP Servers**, **MCP Clients**, **MCP Agents**, and **MCP Apps** in Python and TypeScript.
